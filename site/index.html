Verdict language

A bounded signal, not a security certification.

BLOCK
Errors need attention before configuration is installed or shared.
REVIEW
Warnings or coverage limits still need a human decision.
PASS
No current deterministic rule or known coverage gap matched.

A PASS is not proof of safety. It is a bounded preflight result that still deserves review of high-impact behavior.

What the audit makes visible

Three outputs, one reviewable chain of evidence.

The point is not to infer intent. It is to make the configuration relationships that can affect an agent’s real authority inspectable before execution.

01

Effective Instruction Graph

Traces which project instructions apply at a directory and calls out when a nearer instruction changes an ancestor rule.

“Which instructions are effective here?”

02

Codex Plugin Contract Audit

Checks the declared plugin and skill structure against the supported contract so broken or unsafe references are visible before use.

“Does this plugin and its skills meet the contract?”

03

Capability-to-Authority Map

Connects evidence-backed declarations across instructions, skills, plugins, and MCP configuration without claiming runtime behavior or authorization grants.

“Which declared capabilities connect the configuration?”

Report, not dashboard

A ledger that keeps evidence next to the conclusion.

The self-contained report pairs verdict totals with instruction precedence, capability relationships, and expandable source evidence. It stays legible offline and retains relative paths rather than personal machine paths.

Reproduce locally

Run the fake fixture in your own terminal.

This command produces portable local evidence from the included deliberately unsafe fixture. It is expected to BLOCK, and --fail-on never keeps the demo command successful so the report can be opened.

From the repository root

agentlint scan examples/unsafe-project --json reports/unsafe.json --html reports/unsafe.html --fail-on never
