# OpenAI Build Week submission checklist

Official rules checked: <https://openai.devpost.com/rules>. This checklist is operational guidance, not a substitute for the live rules.

## Before submission

- [ ] Select **Developer Tools** track.
- [ ] Record Windows verification and complete independent macOS and Linux runtime verification before submission; current independent acceptance evidence is Windows (Python 3.14).
- [ ] Run `pytest`, plugin validator, skill quick validator, safe fixture PASS, and unsafe fake-fixture scan.
- [ ] Make the code repository public with the MIT license, **or** share the private repository with `testing@devpost.com` and `build-week-event@openai.com`.
- [ ] Confirm the README explains installation, supported platforms, test path, limitations, and how Codex/GPT-5.6 were used.
- [ ] Record a public YouTube demo under three minutes, with audible English narration, a working product demo, and an explanation of Codex/GPT-5.6 usage.
- [ ] Confirm no video frame exposes real secrets, private data, or unlicensed copyrighted media.
- [ ] Use only repository-portable `examples/unsafe-project` artifacts/screenshots for the public demo or submission; do not upload a personal scan output containing real local paths.
